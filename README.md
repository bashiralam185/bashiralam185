<details>
<summary>🛒 Retail Demand Forecasting — Architecture Deep Dive</summary>
<br/>

**What I built:**
- Multi-level forecasting system covering sales, orders, and demand signals
- End-to-end pipeline deployed for real retail clients

```
POS Data → Feature Engineering → Multi-Level Forecast Models → REST API → Business Teams
```

**Key decisions:**
- Chose a hierarchical forecasting approach to maintain consistency across levels
- Designed the API layer to be consumption-friendly for non-technical business users
- Built modular pipeline so individual components can be updated independently

**Impact:** Improved forecasting accuracy · Enabled data-driven inventory planning · Reduced overstock/stockout costs

</details>

<details>
<summary>🧪 Multimodal AI Benchmarking — Architecture Deep Dive</summary>
<br/>

**What I built:**
- Scalable benchmarking pipeline designed for Vision-Language Models (VLMs)
- Triton Inference Server integration for high-throughput evaluation
- Scenario-based evaluation engine for testing real-world edge cases

```
Image Input → Scenario Engine → VLM Inference (Triton) → Evaluation Metrics → Reports
```

**Key decisions:**
- Triton was chosen over vanilla serving for better GPU utilization and throughput
- Scenario-based design allows non-ML teams to define test cases in plain language

**Impact:** Better model selection decisions · Caught failure modes before production deployment

</details>

<details>
<summary>🏥 Medical OCR Intelligence — Architecture Deep Dive</summary>
<br/>

**What I built:**
- OCR + NLP pipeline to extract structured data from medical PDFs
- Validation framework to catch extraction errors before database write

```
PDF → OCR Engine → NLP Structuring → Validation Layer → Healthcare Database
```

**Key decisions:**
- Added a validation step specifically to handle noisy OCR output from scanned documents
- NLP layer trained on medical terminology for higher extraction accuracy

**Impact:** Automated previously manual digitization workflows · Reduced data entry errors

</details>

<details>
<summary>🚖 Mobility Demand Prediction — Architecture Deep Dive</summary>
<br/>

**What I built:**
- Spatio-temporal demand forecasting model combining time and geolocation signals
- Hotspot map generation for driver allocation recommendations

```
Timestamp + GPS Signals → Spatio-Temporal Features → Demand Model → Hotspot Recommendations
```

**Key decisions:**
- Combined time-series patterns with geographic clustering to improve precision
- Output designed for ops dashboards, not just model outputs

**Impact:** More efficient driver deployment · Improved supply-demand matching

</details>
